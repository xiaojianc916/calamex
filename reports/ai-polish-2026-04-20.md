## 已修复项

### 1. 实时诊断并发积压与旧请求竞态

- 文件：src/components/editor/SmartScriptEditor.vue
- 现象：快速输入时会连续触发多次 ShellCheck IPC，请求虽然会忽略旧结果，但旧调用仍在后台执行，容易把新输入的诊断延后。
- 根因：前端只有 debounce 和“旧结果丢弃”，没有单飞控制；同一时刻可能存在多个 analyzeScript 调用同时飞行。
- 修法：把诊断改为单飞队列。任意时刻只允许一个请求在途，输入继续变化时只保留最新一次待执行请求；卸载后直接丢弃后续返回。
- 验证：npm run lint 通过；npm run tauri:build -- --no-bundle 通过；dev 与 release 冒烟启动通过。

### 2. 临时脚本命名碰撞导致的数据正确性风险

- 文件：src-tauri/src/commands/mod.rs
- 现象：实时诊断临时文件和未保存脚本的终端临时文件都只用“文件名 stem + 毫秒时间戳”命名，在高频输入或极短时间内重复运行时存在同名覆盖风险。
- 根因：文件名唯一性只依赖毫秒级时间戳，Windows 上并发请求落在同一毫秒并不罕见。
- 修法：新增全局原子序列号，与高精度时间戳共同组成临时文件后缀；同时统一使用目标临时目录，不再退回 temp 根目录。
- 验证：npm run tauri:build -- --no-bundle 通过，release 可执行文件成功生成到 target/release/sh-editor.exe。

### 3. 运行日志页隐藏时仍保留重组件挂载

- 文件：src/components/workbench/RunPanel.vue
- 现象：运行日志页在非激活状态下仍通过 v-show 保持挂载，StructuredRunInsights 的缓存、watch 与重建逻辑会长期存在。
- 根因：日志面板使用 v-show，只隐藏不卸载。
- 修法：把日志容器改为 v-if，仅在日志 Tab 激活时挂载 StructuredRunInsights；终端页仍保持挂载，避免误伤终端会话生命周期。
- 验证：npm run lint 通过；npm run tauri:build -- --no-bundle 通过；dev 冒烟启动后能正常起到 debug 桌面进程。

### 4. 终端拖拽在失焦场景下的监听残留风险

- 文件：src/layouts/AppShellLayout.vue
- 现象：拖拽终端分隔条时，如果窗口在 mouseup 前失焦，旧的 mousemove / mouseup 监听可能滞留到下一次拖拽结束才清理。
- 根因：拖拽清理只绑定在 mouseup，没有覆盖 blur；新拖拽开始前也没有主动回收历史 cleanup。
- 修法：新拖拽开始前先执行旧 cleanup；同时在 window blur 时强制执行 stopResize，统一移除 mousemove、mouseup、blur 监听。
- 验证：npm run lint 通过；npm run tauri:build -- --no-bundle 通过。
